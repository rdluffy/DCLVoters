What is DCL-Voters ?

DCL-Voters aims to provide a voting system supporting the "majority judgement" core system using P2P techniques to enable the share of trust among the nodes.

What is the "majority judgement" ?

“Majority judgment is a single-winner voting system proposed by Michel Balinski and Rida Laraki. Voters freely grade each candidate in one of several named ranks, for instance from "excellent" to "bad", and the candidate with the highest median grade is the winner. If more than one candidate has the same median grade, a tiebreaker is used which sees the "closest to median" grade. Majority judgment can be considered as a form of Bucklin voting which allows equal ranks.” @WikiPedia/Majority_judgment.

How does the system aim to work ?

  • The concept behind DCLVoters is to use P2P technology to ensure that no datacenter is needed. Each node/application will have the ability to ensure the system management
  • The nodes are be typical smartphone applications
  • History will be ensure by dedicated nodes

Current Status

  • A POC is being developped in order to set up the technology
    • Perimeter : Android
    • Objective #1 : Manage voting features
    • Objective #2 : Enabling data replication over the nodes
    • Objective #3 : Enabling security

Vocabolary

  • Ballot : It manages the voting systems
  • Candidate : A Ballot contains candidates. The voters will judge the candidates

Voting features

  • User management
    • Creating, Login, Notifying
  • Ballot management
    • Creating, Accessing, Starting, Stopping, Deleting
  • Expression management
    • Expressing a choice, Retrieving Results
